## Phase 2: Research (completed)

### 3 Research Agents Launched
1. **Scraping alternatives** — Playwright remains best for browser automation
2. **Alaska calendar API** — Found `searchbff/V3/search` endpoint (reverse-engineered from AwardWiz)
3. **seats.aero API** — $9.99/mo, 1000 req/day, deferred for MVP

### Key Finding
- Direct curl to `searchbff/V3` returns HTML (Akamai bot protection) — Playwright required
- Industry research: seats.aero (Go), Roame (Go+Firebase), Point.me (Node)

### Language Evaluation
- Evaluated: Rust, Python, C++, Go
- **Go wins:** industry-aligned (seats.aero/Roame), goroutines = natural actors, playwright-go available

## Phase 3: Architecture Design (completed)

### Final Tech Stack
| Layer | Technology | Hosting |
|-------|-----------|---------|
| Frontend | Next.js 16, React, Tailwind, TanStack Query | Self-hosted |
| Scraper | Go, playwright-go, goroutines+channels | Self-hosted |
| Message Bus + Cache | Redis (local) | Local Redis |
| Database | Postgres, pgx, Drizzle ORM | Supabase (free, pooler: aws-0-us-west-2) |

### Key Architecture Decisions (15 total in decisions.md)
- FE reads Redis/Postgres directly, scraper is fully decoupled
- Redis Streams for job queue, Pub/Sub for results
- Write-through: every scrape → Redis cache (60s TTL) + Postgres (permanent)
- Actor-based scraper: Supervisor → Dispatcher → Pool → Workers → Processor
- Anti-bot: UA rotation, Gaussian jitter, exponential backoff, circuit breaker, blocked domains
- Project renamed: alaska → fareclaw → **PointClaw** (`pointclaw/`)
- Go module: `github.com/darrenfu/pointclaw/scraper`
